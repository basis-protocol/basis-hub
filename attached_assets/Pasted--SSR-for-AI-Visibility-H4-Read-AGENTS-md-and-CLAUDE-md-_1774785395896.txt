# SSR for AI Visibility (H4)

Read AGENTS.md and CLAUDE.md first.

## Problem

When ChatGPT, Perplexity, Claude, Google, or any bot/crawler fetches `basisprotocol.xyz`, they get an empty `<div id="root"></div>`. The React SPA only renders in browsers with JavaScript. AI assistants can’t cite Basis scores. Search engines can’t index rankings. The individual entity pages (`/wallet/{address}`, `/asset/{symbol}`, `/pulse/{date}`) already work — they’re server-rendered by `app/publisher/page_renderer.py` with full HTML + JSON-LD. The root dashboard is the gap.

## Solution

Add a bot-detection layer to the SPA catch-all. When a bot/AI hits `/`, serve a server-rendered HTML page with real score data and JSON-LD. Browsers still get the React SPA. Same pattern the existing page_renderer uses.

## Implementation

### Step 1: Bot detection helper

Add this near the top of `app/server.py` (near the other utility functions):

```python
import re

BOT_UA_PATTERN = re.compile(
    r"(bot|crawl|spider|slurp|Googlebot|Bingbot|DuckDuckBot|Baiduspider|YandexBot"
    r"|facebookexternalhit|Twitterbot|LinkedInBot|Discordbot"
    r"|ChatGPT|GPTBot|Claude|ClaudeBot|Anthropic|PerplexityBot|Perplexity"
    r"|Cohere|cohere-ai|Google-Extended|CCBot|amazonbot"
    r"|python-requests|httpx|axios|curl|wget|fetch|Go-http-client"
    r"|Applebot|ia_archiver|archive\.org_bot)",
    re.IGNORECASE
)

def _is_bot(request) -> bool:
    ua = request.headers.get("user-agent", "")
    # Bots, crawlers, AI assistants
    if BOT_UA_PATTERN.search(ua):
        return True
    # No user-agent at all (programmatic)
    if not ua or len(ua) < 10:
        return True
    # Explicit JSON preference (API clients)
    accept = request.headers.get("accept", "")
    if "application/json" in accept and "text/html" not in accept:
        return True
    return False
```

### Step 4: Wire into the SPA catch-all

In `_register_spa_catch_all` (around line 2837), modify the `serve_spa` function to check for bots:

Replace the entire function body:

```python
def _register_spa_catch_all(app_instance):
    """Register the SPA catch-all AFTER all other routes so it doesn't shadow them."""
    @app_instance.get("/{full_path:path}")
    async def serve_spa(request: Request, full_path: str):
        if full_path.startswith("api/") or full_path.startswith("docs") or full_path.startswith("openapi") or full_path.startswith("admin") or full_path.startswith("developers"):
            raise HTTPException(status_code=404, detail="Not found")
        
        # Serve server-rendered HTML for bots/AI on key routes
        if _is_bot(request):
            try:
                if full_path in ("", "/"):
                    return HTMLResponse(
                        content=_render_rankings_html(),
                        headers={"Cache-Control": "public, max-age=300", "Basis-URL-Stability": "permanent"}
                    )
                elif full_path == "witness":
                    return HTMLResponse(
                        content=_render_witness_html(),
                        headers={"Cache-Control": "public, max-age=300", "Basis-URL-Stability": "permanent"}
                    )
            except Exception as e:
                logger.warning(f"SSR render failed for /{full_path}: {e}")
                # Fall through to SPA
        
        index_path = os.path.join(FRONTEND_DIR, "index.html")
        if os.path.exists(index_path):
            return FileResponse(index_path, headers={"Cache-Control": "no-cache"})
        return {"name": "Basis Protocol API", "version": FORMULA_VERSION, "docs": "/docs"}
```

Note: Import `_DecimalEncoder` from `app.publisher.page_renderer` at the top of server.py, OR copy the tiny class into server.py directly since it’s just 4 lines:

```python
class _DecimalEncoder(json.JSONEncoder):
    def default(self, obj):
        from decimal import Decimal
        if isinstance(obj, Decimal):
            return float(obj)
        return super().default(obj)
```
