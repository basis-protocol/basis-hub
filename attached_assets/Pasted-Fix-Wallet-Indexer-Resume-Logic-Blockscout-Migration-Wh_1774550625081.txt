Fix Wallet Indexer: Resume Logic + Blockscout Migration
What & Why
Two changes in one deploy, both touching the same files:

Resume logic. The indexer currently re-scans every wallet from scratch on every run. If the process restarts (Replit redeploy, crash, Etherscan rate limit), it starts over and burns through the entire API quota again. We burned 374K Etherscan calls on a single run because of restarts. Fix: skip wallets already indexed today.
Blockscout migration. Switch from Etherscan V2 API to Blockscout PRO API. Same endpoint patterns (module=token&action=getTokenHolders, module=account&action=tokenbalance), different base URL, better pricing ($0-49/mo vs $199/mo), multi-chain from one key. We'll keep Etherscan as an env-var fallback for one month.

Done looks like

Running the pipeline twice in one day: second run skips all wallets indexed in the first run and completes in seconds with near-zero API calls.
If the pipeline crashes mid-run and restarts, it picks up from where it stopped — only scanning wallets that don't have a last_indexed_at timestamp from today.
The seeding step (fetching top holders) only runs when explicitly requested or when the wallets table has fewer than 1000 entries — not on every incremental run.
All Etherscan API calls go through Blockscout PRO API instead, controlled by env vars.
Setting BLOCK_EXPLORER_PROVIDER=etherscan in env falls back to Etherscan (safety net).

Out of scope

Changing the scoring logic, wallet risk computation, or backlog pipeline
Changing the database schema
Changing any API endpoints in server.py
Touching any spoke repos

Key differences to handle:

Etherscan uses chainid as a query param. Blockscout PRO API v2 also accepts chain_id as a query param in the same route format. Use chain_id=1 for Blockscout.
Etherscan's tokenholderlist → Blockscout's getTokenHolders (different action name, same module=token). The response shape is similar but field names may differ — Etherscan returns TokenHolderAddress, Blockscout returns address. Handle both.
Etherscan's tokenbalance and tokenbalancemulti — verify these exist on Blockscout. If tokenbalancemulti doesn't exist on Blockscout, fall back to individual tokenbalance calls with appropriate rate limiting.
The API key env var: read BLOCKSCOUT_API_KEY when using Blockscout, ETHERSCAN_API_KEY when using Etherscan.

Change to:
python# Check if this is a full re-seed or an incremental run
force_reseed = os.environ.get("INDEXER_FORCE_RESEED", "false").lower() == "true"
existing = _get_existing_wallets()

if force_reseed or len(existing) < 1000:
    # Full seed: fetch top holders from explorer API
    logger.info("Full seed run — fetching top holders from explorer")
    wallet_addresses = await seed_wallets(client, api_key, holders_per_coin)
    wallet_addresses |= existing
else:
    # Incremental run: only re-index existing wallets, no new seeding
    logger.info(f"Incremental run — {len(existing)} existing wallets, skipping seed")
    wallet_addresses = existing

# Filter to wallets not yet indexed today
already_indexed_today = set()
rows = fetch_all("""
    SELECT address FROM wallet_graph.wallets
    WHERE last_indexed_at IS NOT NULL
      AND last_indexed_at >= CURRENT_DATE
""")
already_indexed_today = {row["address"] for row in rows}

wallet_list = [w for w in wallet_addresses if w not in already_indexed_today]
skipped = len(wallet_addresses) - len(wallet_list)

if skipped > 0:
    logger.info(f"Skipping {skipped} wallets already indexed today, {len(wallet_list)} remaining")

if not wallet_list:
    logger.info("All wallets already indexed today — nothing to do")
    return {
        "wallets_discovered": len(wallet_addresses),
        "wallets_indexed": 0,
        "wallets_skipped_today": skipped,
        "message": "All wallets already indexed today",
    }
