PROBLEM

The Daily Pulse page at basisprotocol.xyz/pulse shows incorrect data:

1. AVG RISK displays 0.0
2. WALLETS displays 44,697 (total ever discovered, not actively scored)
3. EVENTS 24H shows 200 events, all classified as "silent"

ROOT CAUSE ANALYSIS

Bug 1: AVG RISK = 0.0

The pulse generator (app/pulse_generator.py, lines 71-77) runs:

SELECT COUNT(DISTINCT wallet_address) as wallets_scored,
       COALESCE(AVG(risk_score), 0) as avg_risk_score
FROM wallet_graph.wallet_risk_scores
WHERE computed_at > NOW() - INTERVAL '48 hours'

The risk_score column is NULL when a wallet has zero scored holdings (app/indexer/scorer.py, lines 47-56). If is_scored is false for all holdings, risk_score stays NULL. AVG of all NULLs = NULL, then COALESCE(NULL, 0) = 0.

DIAGNOSTIC — RUN THESE FIRST:

SELECT is_scored, COUNT(*) FROM wallet_graph.wallet_holdings GROUP BY is_scored;

SELECT COUNT(*) as total, COUNT(risk_score) as non_null, COUNT(*) - COUNT(risk_score) as null_scores
FROM wallet_graph.wallet_risk_scores
WHERE computed_at > NOW() - INTERVAL '48 hours';

SELECT stablecoin_id, symbol, contract FROM stablecoins WHERE contract IS NOT NULL AND contract != '';

SELECT COUNT(*), MIN(computed_at), MAX(computed_at) FROM wallet_graph.wallet_risk_scores;

Two possible scenarios:

Scenario A — rows exist but risk_score is NULL: The scanner finds holdings and marks is_scored=true, but sii_score is NULL even when is_scored=true. This means the scores table stablecoin_id values don't match what get_all_known_contracts() returns. Check app/indexer/config.py line 21 — it keys by contract address, and the stablecoin_id comes from STABLECOIN_REGISTRY. Then in scanner.py line 142-143, it does sii_scores.get(sid) where sii_scores comes from _get_current_sii_scores() which keys by stablecoin_id from the scores table. If those IDs don't match, sii_score stays None even though is_scored is True.

Scenario B — no recent rows at all: The re-indexing pipeline in main.py (lines 146-156) runs app/indexer/pipeline.run_pipeline() every 24h. Check Replit logs for "Scheduled wallet re-indexing" messages. If it's erroring silently, wallet_risk_scores never gets populated.

Replace the wallet stats section (lines 70-85) with:

Bug 3: All events silent

Run this diagnostic:

    SELECT trigger_type, severity, COUNT(*)
    FROM assessment_events
    WHERE created_at > NOW() - INTERVAL '48 hours'
    GROUP BY trigger_type, severity
    ORDER BY COUNT(*) DESC;

    SELECT wallet_address, COUNT(*) as assessments
    FROM assessment_events
    GROUP BY wallet_address
    HAVING COUNT(*) > 1
    LIMIT 10;

If all events are trigger_type = 'daily_cycle' with only 1 assessment per wallet, this is expected cold-start — the classifier in app/agent/classifier.py (line 88-90) computes score_delta from current vs previous wallet_risk_score, and with no previous assessment everything falls to silent. The second daily cycle should produce deltas.

If wallets DO have history but events are still all silent, check whether wallet_risk_score is NULL in the assessments (same root cause as Bug 1 — if risk_score is NULL, score_delta = 0, everything is silent).
