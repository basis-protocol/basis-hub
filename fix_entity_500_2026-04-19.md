# Fix: /entity/{slug} 500 — 2026-04-19

## Root Cause

**Line 976 of app/server.py:** Invalid f-string format specifier.

```python
# BROKEN — Python parses ".1f if score else '—'" as the format spec for score
f"{score:.1f if score else '—'}"

# FIXED — nested f-string evaluates the conditional first
f'{f"{score:.1f}" if score else "—"}'
```

`ValueError: Invalid format specifier '.1f if score else '—'' for object of type 'float'`

This crashes on EVERY entity page because the score value always exists (it's fetched from the DB). The f-string format spec parser doesn't support conditional expressions after the colon — everything after `:` is treated as the format string literally.

## File:Line

`app/server.py:976` — the HTML template's score display div.

## Fix Applied

Single line change. No structural changes to the handler.

## Test

```python
score = 89.2
# Crashes:
f"{score:.1f if score else '—'}"  # ValueError

# Works:
f'{f"{score:.1f}" if score else "—"}'  # "89.2"
```
