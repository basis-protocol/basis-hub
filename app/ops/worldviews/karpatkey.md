# karpatkey (kpk)

## Worldview Summary

Policy-defined execution. Permissions as enforcement on Safe/Zodiac. Deterministic agents within predefined bounds. Investment Policy Statements as governance standard. DeFi Treasury Network covering every primitive. Stablecoin quality is still ad-hoc across their DAO network ($2.1B+ in managed treasuries across ENS, CoW, Lido, Aave, Uniswap, Safe).

## Key Themes

- **Policy as code:** Everything operates within predefined bounds. Governance legitimacy comes from transparent, deterministic policy enforcement.
- **Permissions as enforcement:** Safe + Zodiac modules create on-chain guardrails. Agents can only act within what the IPS permits.
- **Investment Policy Statements (IPS):** The canonical governance artifact. Defines what a treasury can and can't hold, with specific allocation parameters.
- **Networked governance:** DeFi Treasury Network provides shared infrastructure across DAOs. What benefits one benefits all.
- **Deterministic execution:** No discretionary allocation. Policy parameters are set by governance, execution is mechanical.

## Bridge Patterns

- "Stablecoin quality as IPS input" — the IPS defines allocation rules, but the quality assessment of each stablecoin is still subjective/ad-hoc. SII provides the missing standardized input.
- "Missing input to permissions layer" — the Zodiac module enforces rules, but the stablecoin quality dimension of those rules has no data feed.
- "Unstandardized layer in deterministic execution" — everything else in the karpatkey stack is deterministic. Stablecoin selection is the one judgment call.
- "Network-wide quality standard" — they manage treasuries for 6+ DAOs. Each DAO's stablecoin assessment is independent. SII provides the shared layer.

## Language to Use

- policy, permissions, curation, governance legitimacy, deterministic
- IPS input, quality signal, standardized assessment
- network-wide, shared infrastructure, transparent methodology

## Language to Avoid

- risk infrastructure (too generic, sounds like a competitor)
- neutral standard (implies they need correction)
- primitive, protocol (too abstract for their operational frame)
- "replace" or "automate" (they value their expert judgment)
