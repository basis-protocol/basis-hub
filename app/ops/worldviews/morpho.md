# Morpho

## Worldview Summary

Curators are the risk management layer. Permissionless lending. Curation encodes strategy through allocation. Institutional-grade risk is the differentiator. The protocol doesn't assess risk — curators do. This division of labor is a design principle, not a limitation.

## Key Themes

- **Curation as risk management:** Curators set allocation parameters, choose which markets to include. The protocol is neutral infrastructure; curators are the intelligence layer.
- **Permissionless by design:** Anyone can create markets. The quality filter is curation, not protocol gatekeeping.
- **Institutional-grade ambition:** Morpho Effect monthly reports, structured curator onboarding, emphasis on institutional adoption.
- **The curator ecosystem:** Multiple curators (Re7, 9summits, Steakhouse, MEV Capital, Gauntlet, Sentora, Bitwise) each with different strategies. Ecosystem health depends on curator quality.

## Bridge Patterns

- "Shared input to curator decisions" — each curator independently evaluates stablecoin quality. SII provides a shared baseline that curators can build on.
- "Pre-allocation quality check" — before a curator allocates to a stablecoin market, what's the quality signal? Currently each curator answers this differently.
- "Curator differentiation layer" — curators compete on strategy. SII scores are a shared data layer that doesn't reduce differentiation but raises the floor.
- "The Resolv precedent" — the Resolv incident affected multiple curators simultaneously. A shared quality signal would have flagged the deterioration before allocation.

## Language to Use

- curation, allocation, permissionless, curator ecosystem
- shared data layer, pre-allocation input, market quality
- institutional-grade, risk visibility

## Language to Avoid

- gatekeeper (antithetical to permissionless philosophy)
- risk rating (too centralized-sounding)
- mandatory, required (curators value autonomy)
